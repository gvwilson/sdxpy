--- title: "Matching Patterns" template: slides --- ## The Problem - Match text with patterns - Initially handle just five cases | Meaning | Character | | ------------------------- | --------- | | Any literal character *c* | *c* | | Any single character | . | | Beginning of input | ^ | | End of input | $ | | Zero or more of something | * | --- ## Start With Something Simple - If the first element of the pattern matches where we are, see if the rest of the pattern matches what's left - Otherwise, see if the the pattern will match further along --- ## Overall Matcher ```python def match(pattern, text): # Empty pattern matches any string. if not pattern: return True # Start of string. if pattern[0] == "^": return match_here(pattern, 1, text, 0) # Need 'do-while' to handle zero characters matching. i_text = 0 while True: if match_here(pattern, 0, text, i_text): return True i_text += 1 if i_text == len(text): break # Nothing worked. return False ``` --- ## Match at Location ```python def match_here(pattern, i_pattern, text, i_text): if no_more_pattern(pattern, i_pattern, text, i_text): return True if dollar_at_end_of_text(pattern, i_pattern, text, i_text): return True if supposed_to_match_many(pattern, i_pattern, text, i_text): next_i_text = match_many(pattern, i_pattern, text, i_text) return match_here(pattern, i_pattern+2, text, next_i_text) if no_more_text_to_match(pattern, i_pattern, text, i_text): return False if match_single_character(pattern, i_pattern, text, i_text): return match_here(pattern, i_pattern+1, text, i_text+1) return False # nothing worked ``` -- - Meaningful function names help a bit --- ## A Few Cases ```python def no_more_pattern(pat, i_pat, text, i_text): return i_pat == len(pat) def match_single_character(pat, i_pat, text, i_text): return (pat[i_pat] == ".") \ or (pat[i_pat] == text[i_text]) def supposed_to_match_many(pat, i_pat, text, i_text): return ((len(pat) - i_pat) > 1) \ and (pat[i_pat + 1] == "*") def match_many(pat, i_pat, text, i_text): while (i_text < len(text)) and (text[i_text] == pat[i_pat]): i_text += 1 return i_text ``` --- ## Testing ```python def test_simple_regexp(): tests = [ ["", "", True], ["a", "a", True], ["b", "a", False], ["a", "ab", True], ["b", "ab", True], ["ab", "ba", False], ["^a", "ab", True], ["^b", "ab", False], ["a$", "ab", False], ["a$", "ba", True], ["a*", "", True], ["a*", "baac", True], ["ab*c", "ac", True], ["ab*c", "abc", True], ["ab*c", "abbbc", True], ["ab*c", "abxc", False], ] for (regexp, text, expected) in tests: assert match(regexp, text) == expected ``` --- ## Butâ€¦ - Doesn't work - Think about matching `a*ab` against the string `aab` - And it will be very difficult to extend - Are you *sure* the clauses in `match_here` are in the right order? --- ## A Different Approach - Use the __Chain of Responsibility__ design pattern - Each object matches what it can and passes the rest down the chain --- ## Looks a Lot Like Lists - The expression to match `ab*c` is: ```python Lit( "a", Any( Lit("b", None), Lit("c", None) ) ) ``` --- ## Base Class ```python class RegexBase: def __init__(self, rest=None): self.rest = rest def match(self, text): for i in range(len(text) + 1): if self._match(text, i) is not None: return True return False def _match(self, text, start): raise NotImplementedError("must override '_match'") ``` - `RegexBase.match` is the overall strategy - Start here and keep trying if you need to - The `_match` method has the matcher-specific logic - Returns the next index or `None` --- ## Matching Literal Characters ```python class Lit(RegexBase): def __init__(self, chars, rest=None): super().__init__(rest) self.chars = chars def _match(self, text, start): next_index = start + len(self.chars) if next_index > len(text): return None if text[start:next_index] != self.chars: return None if self.rest is None: return next_index return self.rest._match(text, next_index) ``` --- ## Match the Start of Text ```python class Start(RegexBase): def _match(self, text, start): if start != 0: return None if not self.rest: return 0 return self.rest._match(text, start) ``` -- - Isn't that simpler? -- - But yes, it could use some comments --- ## Match Zero or More ```python class Any(RegexBase): def __init__(self, child, rest=None): super().__init__(rest) self.child = child def _match(self, text, start): max_possible = len(text) - start for num in range(max_possible, -1, -1): after_many = self._match_many(text, start, num) if after_many is not None: return after_many return None def _match_many(self, text, start, num): for i in range(num): start = self.child._match(text, start) if start is None: return None if self.rest: return self.rest._match(text, start) return start ``` --- ## Is This Better? - Solve one matching problem at a time - Which also makes it easy to extend - Not a coincidence that we get both of these together - But distributed decision making can be harder to follow --- class: summary ## Summary --- class: exercise ## Find and Fix the Error The first regular expression matcher contains an error: the pattern `'a*ab'` should match the string `'aab'` but doesn't. An alternative is __lazy matching__, in which expressions match as little as they need to. For example, given the string `"ab"`, an eager match with the expression `ab*` will match both letters (because `b*` matches a 'b' if one is available) but a lazy match will only match the first letter (because `b*` can match no letters at all).
