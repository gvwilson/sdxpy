--- ## Who You Are - **Aïsha** started writing Python in an undergrad geology course. After spending grad school creating ever-more-complex Jupyter notebooks she wants to learn how to build packages other people can use. - **Rupinder** is a computer science undergrad. He has learned a lot of theory, and uses Git and unit testing tools in assignments, but doesn't understand how they work. - **Yim** teaches courses on full-stack web development and software design. They are happy with the former, but frustrated that books about the latter are so abstract and use examples that their students can't relate to. --- ## What You Will Get Out of This - This material will fill in some gaps in **Aïsha's** programming knowledge and teach her some common design patterns. - This material will give **Rupinder** a better understanding of his tools and show him how to design new ones. - This material will give **Yim** starting points for a wide variety of course assignments. --- ## You Should Already Know… - How to use dictionaries, exceptions, and classes. - We assume that if you can use these, you can also use lists, loops, conditionals, and functions. - Create static web pages using HTML and CSS. - Use Git to save and share files. - It's OK not to know the more obscure commands. - Process nested lists recursively. - This is the most complicated thing we *don't* explain. --- class: aside ## Pedagogical Notes - Diagrams help (most) people learn [Wilson2019] -
